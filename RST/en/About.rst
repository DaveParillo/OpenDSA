About this course
=================
This course introduces students to data structures and object-oriented 
software engineering. Emphasis is placed on basic data structures, including 
collections and linked structures (arrays, lists, stacks, queues, trees, 
and hashes) from the perspective of object-oriented implementation. 

Topics also include object-oriented analysis, design, and implementation 
in popular programming languages, such as C++, C#, and Java. 

Upon successful completion of the course the student will be able to:

#. Compare, contrast and apply the concepts of the main phases (disciplines) 
   of software engineering best practices, including an iterative approach to 
   project management to form the contextual foundation for syntax, logic, and 
   semantics learning of data structures, classes, and objects.
#. Identify and apply the language features in C, C++, and/or Java to the 
   development of reusable software components, using principles of abstraction, 
   inheritance, encapsulation, and polymorphism.
#. Identify and apply various data structures in C++, C#, and/or Java to the 
   development of useful, robust, and reusable software components.
#. Apply relevant Design Patterns to the best-practices implementation of 
   data structure components through the correlation of their understanding 
   of object-orientation with the implementation of various data structures in 
   C-based object oriented (O-O) languages.
#. Demonstrate and practice the serialization of objects for persistent storage 
   in files and databases to handle issues of file and database input-output, 
   including opening and closing, connecting, and read and write operations.
#. Identify and apply the various approaches pursued in C-family languages 
   to manage memory, including their respective impacts on program performance 
   and efficiency.


Course Outline and Scope
-------------------------

The following topics are included in the framework of the course but are 
not intended as limits on content. 
The order of presentation and relative emphasis will vary with each instructor.

- Unified Process-based Software Engineering Best Practices

  - Software engineering best practices
  - Design patterns
  - Unified Modeling Language
  - Phases of development and iterations in the Unified Process
  - eXtreme Programming and other agile techniques

- Object-Orientation Overview: 

  - C Languages Syntax, Structure, and Semantics in the Context of O-O
  - Modularity, component reuse
  - Abstraction, inheritance, encapsulation, polymorphism
  - Variables and issues of scope and memory management
  - Syntax and semantics of O-O languages (C languages)

- Data Structures: Syntax and Semantics

  - Primitive types
  - Information organization and management
  - Compile-time versus run-time
  - Parameter passing (by value, by reference)
  - Custom types
  - Collections, stacks/queues, and linked structures

- Data Structures with Classes and Objects

  - Class hierarchy
  - Polymorphic data structues
  - Typing and scope

- Data Persistence

  - Heap and stack
  - File input-output
  - Object serialization
  - Database access

- Memory Management and Performance

  - Allocating and freeing memory
  - Pointers
  - Garbage collection

- Lab activities will include programming assignment and 
  exercises that investigate and practice the key concepts learned in lecture.


Assignments
-----------

Reading Assignments
...................

Reading assignments are required and may include but, are not limited to, the following:

- Weekly chapter reading assignments based on a current textbook covering 
  object-orientation and data structures.
- Online tutorials.




Writing Assignments
...................
Writing assignments are required and may include, but are not limited to, the following:

- Summaries of software program design and documentation.
- Exercises related to program source and object code.
- Short papers related to topics, such as object-oriented software development methodology; 
  data structure syntax and semantics, e.g. arrays, vectors, linked lists, etc.; 
  file I/O, database connectivity; variable scope; and parameter passing.

Portfolio Project
...............................

The portfolio project is a capstone project that deomonstrates understanding
of all of the material presented in the course.  Components of the project
will include:

 - Justification for project selection;
 - Interviews with project stakeholders;
 - Design specifications;
 - Source code and screen shots of running program.
 - Preparation for in-class portfolio presentation.

Appropriate Assignments that Demonstrate Critical Thinking
..........................................................
Critical thinking assignments are required and may include, but are not limited to, the following:

- Language feature comparisons.
- Analysis of the development and use of software and computer technology in the broader context of
  socio-economic issues.
- Portfolio project that requires students to solve real-life business problems using software
  technologies and applications.

Evaluation Methods
------------------
A student's grade will be based on multiple measures of performance unless the course requires no grade.
Multiple measures may include, but are not limited to, the following:

- Programming assignments that test the ability to use the syntax and semantics rules of the
  C-language-family tool chosen for the course (C++, C#, or Java) to solve a particular problem.
- Examinations (at least a midterm and final are recommended) to test their understanding of, and
  ability to, apply O-O design and programming and software engineering concepts.
- Class participation and team work.
- Portfolio project (with presentation) is highly recommended.
